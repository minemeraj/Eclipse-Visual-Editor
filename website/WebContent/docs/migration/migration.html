Migration
Migration for VE Developers and Extenders
 
Description
This is document links to the migration documents needed for VE developers and VE extenders (those that use VE as a base and extend it).
 
  • Migrate to VE 1.2
    • Migrate Palette XMI to new format. It has been restructured to match directly the GEF structure instead of the old structure. For VE 1.2 the old format and new format will continue to be supported. However, do not mix in the same palette xmi file the old format and the new format. If that is done the migration facility will fail. The old palette format will no longer be supported in VE 1.3. So please migrate as soon as possible before VE 1.3. Migration instructions.
    • Migrate Override files from internal Event format to EMF API ChangeDescription format. The override file contents have been changed to use the EMF API ChangeDescription format. This way we are using API's. The old Event format and the new ChangeDescription format will continue to be supported for VE 1.2. The old event format will no longer be supported in VE 1.3. So please migrate as soon as possible before VE 1.3. Migration instructions.
