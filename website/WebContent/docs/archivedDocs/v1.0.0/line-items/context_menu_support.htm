Context Menu Support

Eclipse Popup Extensions

Context menus (or popup menus) are provided via the Eclipse extension point org.eclipse.ui.popupMenus and need to be added for various SWT controls:

  • TabFolder - Switch To - allow switching between the various pages of the notebook
  • Composite - null layout - Show Grid/Hide Grid - show and hide a grid of dots that allow the user to snap controls to a grid position within the composite. This helps with control alignment within the composite.
  • Direct edit capability - setText capability allows the user to change the text of a control without having to find the property in the property sheet viewer.

 

ObjectActionDelegates

IObjectActionDelegate is an Interface for an object action that is contributed into a popup menu for a view or editor. It extends IActionDelegate and adds an initialization method for connecting the delegate to the part it should work with. This are used in conjunction with the Eclipse popup extensions to delegate selections and editor changes to the actions specified.

ObjectActionDelegates need to be implemented for SWT context menu support for each of the popup menus described above.

 

ActionFilters

IActionFilter is an adapter which performs filtering on the actions which appear in the context menu. The visibility of each action is controlled by filtering only those actions that are using a specific object class and optional name filter attributes that are key/value pairs. This prevents irrelevant or invalid actions from showing up in the context menu.

The following filters need to be implemented for SWT context menu support:

  • null layout support for Show/Hide grid
  • GridLayout support for Show/Hide grid
