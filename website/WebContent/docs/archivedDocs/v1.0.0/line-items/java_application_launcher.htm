Java Application Launcher

Currently the Visual Editor provides the capability to launch a visual application using the VE's own Java application launcher by selecting the class from the navigator and selecting the menu item Run-->Run As-->Java Bean

This allows the user to customize various optional parameters, the Swing Look & Feel, and even a different JRE if not the default. This is very specific to AWT/Swing applications in that the class is hosted within a JFrame (Swing) or Frame (AWT) class.

  • This needs to modified such that it will support both SWT and AWT/Swing applications.
  • The launcher needs a way of determining if the selected visual application is an SWT or AWT/Swing application.
  • SWT applications need to be handled in the following manner when launched with the VE Java Launcher:
    • A Visual class that extends Composite should be hosted within temporary Shell and the temporary Shell called from within a main method. Perhaps provide main method code generation if one doesn't exist.
    • A Visual class that extends Object and has a main method will be launched as is.
    • A Visual class that extends Object and doesn't have a main method will be have a main method generated.
  • When the launcher wizard is displayed, the Applet Parameters tab and page, and the Swing Look & Feel parameters should not be shown.
